Build a complete, production-ready, full-stack travel agency web application called 'Rihla' (رحلة) using Next.js 15 (App Router), TypeScript, Prisma ORM, SQLite database, Tailwind CSS, shadcn/ui, Framer Motion, and next-intl for i18n (Arabic/French/English with RTL support). All currency in Algerian Dinars (DZD / دج).
━━━ STACK & TOOLING ━━━
  • Framework: Next.js 14 App Router with TypeScript strict mode
  • Database: SQLite via Prisma with @prisma/client — primary DB
  • ORM: Prisma 5.x with full migration support and seed script
  • Styling: Tailwind CSS 3.x with RTL plugin (tailwindcss-rtl)
  • UI Library: shadcn/ui (Radix primitives, fully accessible)
  • Animation: Framer Motion for page transitions, scroll effects, hover states
  • Charts: Recharts for dashboard stats visualization
  • Auth: NextAuth.js v5 with Credentials provider + JWT sessions
  • i18n: next-intl with locale routing /[locale]/ prefix (ar, fr, en)
  • Forms: React Hook Form + Zod validation (client + server)
  • Tables: TanStack Table v8 (headless, sortable, filterable)
  • State: Zustand for UI preferences (sidebar, language, theme)
  • Email: Nodemailer for transactional emails
  • Linter/Formatter: Biome (replaces ESLint + Prettier) — biome.json config
  • Git Hooks: Husky with pre-commit (biome lint-staged), commit-msg (commitlint Conventional Commits), pre-push (tsc + vitest)
  • Icons: Lucide React
  • Fonts: Inter (Latin) + Noto Naskh Arabic (via next/font/google)
━━━ DATABASE SCHEMA (Prisma) ━━━
  • User: id(cuid), name, email(unique), password(bcrypt), phone?, role(EMPLOYEE|SUPER_ADMIN), avatar?, isActive(default:true), lastLoginAt?, createdAt, updatedAt
  • TravelPackage: id, titleAr, titleFr, titleEn, descAr, descFr, descEn, destination, type(HAJJ|UMRAH|DOMESTIC|INTERNATIONAL), pricePerPerson(Float DZD), durationDays, maxCapacity, images(Json String[]), status(ACTIVE|DRAFT|ARCHIVED), departureDates(Json), inclusions(Json), exclusions(Json), itinerary(Json), rating(Float default 0), createdAt, updatedAt
  • Reservation: id, code(unique RHL-YYYY-NNNNN), clientName, clientEmail, clientPhone, clientNIN(Algerian 18-digit ID), numberOfTravelers, travelDate, totalPriceDZD, status(PENDING|IN_REVIEW|CONFIRMED|REJECTED|CANCELLED|COMPLETED), specialRequests?, rejectionReason?, packageId(FK), assignedToId(FK User?), notes?, createdAt, updatedAt
  • ReservationLog: id, reservationId(FK), userId(FK User?), fromStatus, toStatus, note?, createdAt
  • Testimonial: id, clientName, content, rating(1-5), packageId(FK), isApproved(default:false), createdAt
  • AgencySettings: id(singleton '1'), nameAr, nameFr, nameEn, logo?, address, phone, email, socialLinks(Json), heroTitleAr, heroTitleFr, heroTitleEn, heroSubtitleAr, heroSubtitleFr, heroSubtitleEn
  • ContactMessage: id, name, email, phone?, subject, message, isRead(default:false), createdAt

━━━ RESERVATION FLOW (Client) ━━━
  • 4-step wizard: (1) Package Selection → (2) Personal Info → (3) Travel Details → (4) Review & Confirm
  • Step 1: Package summary display, departure date selector (available dates from DB), travelers count (1-20), live price calculation in DZD
  • Step 2: Full Name, Email, Phone (+213 format), Algerian NIN (18 digits), Special Requests
  • Step 3: Confirm travel date, number of travelers, review total price breakdown (price/person × travelers = total DZD)
  • Step 4: Full summary, terms checkbox, Submit button
  • POST /api/reservations → generate code RHL-YYYY-NNNNN → send confirmation email → show success page with code
  • Zod validation on all fields, inline error messages, progress bar between steps
  • Framer Motion slide animation between steps (slide left on advance, slide right on back)
━━━ AUTH (Login) ━━━
  • Single login page at /[locale]/login for both employees and super admins
  • Email + password form. NextAuth Credentials provider. After login → redirect based on role: EMPLOYEE → /[locale]/employee/dashboard, SUPER_ADMIN → /[locale]/admin/dashboard
  • Middleware: protect /employee/* (EMPLOYEE+) and /admin/* (SUPER_ADMIN only)
  • Forgot password: generate reset token, email link (Nodemailer)
  • Rate limit: 5 failed attempts → 15 min lockout (tracked in DB or memory)
━━━ EMPLOYEE DASHBOARD ━━━

  • HOME: Welcome banner with employee name. 4 KPI cards: Total Handled, Confirmed, Pending, Rejected (with vs last month %). Pie chart: status distribution. Bar chart: daily reservations this month.

  • PROFILE: Avatar upload, name/phone edit, change password form.
  • STATS PAGE: Personal performance — charts for processed per day/week/month, status breakdown, revenue processed, average handling time, performance score (confirmed/total × 100).
━━━ SUPER ADMIN DASHBOARD ━━━
  • OVERVIEW: 6 KPI cards: Total Reservations, Pending, Confirmed, Revenue DZD (total), Active Packages, Total Employees. Date filter (today/week/month/year). Revenue line chart (time series). Reservations status bar chart. Top 5 packages by bookings. Top 5 employees by performance. Recent reservations feed (live poll every 30s). Geographic heatmap by Algeria wilaya (SVG map).

Create/Edit package form: multilingual fields (tabbed: AR/FR/EN), type, price (DZD), duration, capacity, departure dates picker, image upload (multi), inclusions/exclusions lists, itinerary builder (day-by-day accordion). Archive/restore packages. Duplicate package action.
  • RESERVATIONS (GLOBAL): Same as employee but sees ALL. Extra: assign to employee dropdown. Bulk actions: confirm all selected, reject all (with reason). CSV export with date range filter.
  • SETTINGS: Agency profile form. Hero content per language. Featured packages selector. Testimonials approval (list with approve/reject). Contact messages inbox (mark read/unread).
  • AUDIT LOG: Table of all ReservationLogs — timestamp, employee, from status, to status, note. Filterable.
